MeetBrief / Setup

Up and running
in under 5 minutes

MeetBrief is an Electron app. If you have Node.js and an API key, you're ready. No installers, no accounts, no signup — just clone and run.

🍎 macOS 🪟 Windows 🐧 Linux
Step 1

Prerequisites

⬢
Node.js 18 or newer Download from nodejs.org or install via nvm. Run node --version to check.
📦
npm (comes with Node) No separate installation needed — npm is bundled with every Node.js release.
🔑
API key (Anthropic or Ollama) For Claude: get an API key at console.anthropic.com. For fully local use, install Ollama and pull a model. No key needed for Ollama.
🍎
macOS: Screen Recording permission System audio capture requires Screen Recording access. On first launch, MeetBrief will prompt for it automatically via System Preferences → Privacy & Security.
Step 2

Clone and install

MeetBrief lives in the MeetingCopilot/ directory of the TinyTools repository.

1

Clone the repository

The MeetingCopilot/ subfolder contains the full Electron app. You only need this subfolder.

2

Enter the directory

Navigate into MeetingCopilot/ — this is where package.json and the Electron entry point live.

3

Install dependencies

npm will fetch Electron, @huggingface/transformers, and other runtime dependencies.

terminal 
# Clone the full repo
git clone https://github.com/martian-coder/TinyTools.git

# Enter the app directory
cd TinyTools/MeetingCopilot

# Install all dependencies
npm install
Step 3

Configure your provider

Create a .env file in MeetingCopilot/ and set your provider and API key. Your key stays on your machine — it is never sent to MeetBrief.

Claude setup: Get your API key at console.anthropic.com → API Keys. Paste it below. Your key is billed per-token directly by Anthropic — typically under $0.10/hour of meeting time with prompt caching enabled.
Ollama setup: Install Ollama from ollama.ai, then run ollama pull llama3.1 (or any model). No API key needed. Latency depends on your hardware — a mid-range GPU gives good results.
.env (Claude) 
# LLM provider: anthropic | ollama
LLM_PROVIDER=anthropic

# Your Anthropic API key
ANTHROPIC_API_KEY=sk-ant-api03-...

# Whisper model size: tiny | small | medium
# whisper-small is recommended (good speed/accuracy balance)
WHISPER_MODEL=Xenova/whisper-small
.env (Ollama) 
# LLM provider: anthropic | ollama
LLM_PROVIDER=ollama

# Ollama model name (must be pulled first)
OLLAMA_MODEL=llama3.1

# Whisper model size
WHISPER_MODEL=Xenova/whisper-small
Your API key stays on your machine. It is sent directly from your machine to Anthropic (or your local Ollama instance). It never passes through MeetBrief infrastructure — there is none.

Full .env reference

VariableValuesDescription
LLM_PROVIDER anthropic | ollama Which LLM backend to use
ANTHROPIC_API_KEY sk-ant-... Anthropic API key (required when using Claude)
OLLAMA_MODEL llama3.1, mistral, … Model name to use with Ollama
WHISPER_MODEL Xenova/whisper-tiny
Xenova/whisper-small
Xenova/whisper-medium		 Whisper model size. Small is recommended. Larger = more accurate but slower.
Step 4

Launch MeetBrief

Once dependencies are installed and .env is configured, you're ready to start.

1

First run — Whisper download

On the very first launch, MeetBrief will download the Whisper model (~250MB for small) and cache it locally. This is a one-time operation.

2

Grant permissions (macOS)

macOS will prompt for Screen Recording permission. Grant it — this is required for system audio capture to work. Verify in System Settings → Privacy & Security → Screen Recording.

3

Load context, start session

Paste your meeting context into the text area, choose your provider, and click Start Session. Join your meeting — MeetBrief will begin listening.

terminal 
# Start MeetBrief
npm start

# The app window will open.
# On first run, Whisper downloads (~250MB).
# Accept the Screen Recording prompt on macOS.

Troubleshooting

Common issues

No system audio is being captured
macOS: Check System Settings → Privacy & Security → Screen Recording and ensure MeetBrief is listed and enabled. Windows: Ensure the default playback device is set correctly. Linux: System audio requires a PulseAudio loopback module.
API calls fail with an authentication error
Check that your ANTHROPIC_API_KEY in .env is correct and starts with sk-ant-. Make sure you're in the MeetingCopilot/ directory when running npm start.
Whisper download fails or hangs
The model downloads from Hugging Face CDN (~250MB for whisper-small). If it fails, check your internet connection and try again. The model is cached in ~/.cache/meetbrief/ after the first successful download.
Ollama not connecting
Ensure Ollama is running (ollama serve) and accessible at http://localhost:11434. Verify your model is pulled: ollama list. The model name in OLLAMA_MODEL must exactly match the name shown by ollama list.
