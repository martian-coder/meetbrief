MeetBrief / Privacy

Built for people who
take privacy seriously

MeetBrief was designed with one rule: your data never touches our infrastructure. There is no MeetBrief infrastructure. Audio stays on your machine, API keys go directly to your provider, and there's no backend to receive anything.

✓ No MeetBrief servers ✓ Audio stays on-device ✓ Your API key, your billing ✓ Open source — inspect the code
🔑

Your API key, forever yours

You bring your own Anthropic or other provider API key. It is stored locally in your app's config directory and sent directly from your machine to the provider. MeetBrief reads it, uses it, and never sends it anywhere else.

🖥️

Audio processed on-device

All audio capture and transcription runs on your machine via local Whisper. Your microphone and speaker audio never leaves your computer — not even to a relay server. The only network call is your prompt going to your chosen LLM API.

🚫

Zero recording by us

MeetBrief has no backend, no analytics, no telemetry. There is no server to receive data. The app is fully self-contained. You can verify this by inspecting the source code on GitHub — every network call is visible.

Exactly what goes where

A complete picture of every piece of data MeetBrief handles and where it travels.

What MeetBrief sends — and doesn't send

✓
Meeting transcript → your chosen LLM (Claude API or local Ollama) The live transcript of what was said in your meeting is sent to your LLM provider for processing. For Claude: this goes directly from your machine to api.anthropic.com over HTTPS, authenticated with your own API key, billed to your own account. For Ollama: this goes to localhost — it never leaves your machine at all.
✓
Whisper model downloaded once from Hugging Face CDN On first launch, MeetBrief downloads the Whisper model weights (~250MB) from Hugging Face's CDN. This is a standard model weights download over HTTPS. After download, the model runs entirely locally — no further network calls for transcription.
✗
Audio data → MeetBrief servers (does not happen) Audio captured from your speakers or microphone is processed entirely on-device by Whisper. It is never sent to any server — not Anthropic's, not MeetBrief's (there are none). The raw audio never leaves your machine.
✗
API key → MeetBrief servers (does not happen) Your Anthropic API key is stored in your local app data directory. It is read by MeetBrief only to authenticate requests to api.anthropic.com on your behalf. It is never included in any outbound request to a MeetBrief endpoint.
✗
Analytics or telemetry → any server (does not happen) MeetBrief contains no analytics SDK, no crash reporter, no usage telemetry. There are no background pings, version checks, or heartbeats. The app makes no network calls except the ones you explicitly trigger (LLM API calls and the one-time Whisper download).
Technical details

How the isolation is enforced

API key storage

Your API key is stored in Electron's userData directory on your local filesystem — the same location as app preferences and session history. It is read from disk at startup and used directly for API calls. It is never written to any cloud sync location.

  • Stored in local userData config
  • Never included in any request to meetbrief.io or any third-party
  • Only sent to api.anthropic.com (or localhost:11434 for Ollama)

Audio pipeline

System audio is captured as raw PCM via a native subprocess (macOS) or OS loopback API. It is downsampled, chunked to 100ms frames, and processed entirely in-process by Silero VAD and Whisper. The raw audio buffers are kept in memory and discarded after transcription.

  • No audio written to disk during sessions
  • No audio sent over any network socket
  • Whisper runs via @huggingface/transformers in-process

LLM API calls

Each API call contains your system prompt (meeting context) and the current transcript window — no device identifiers, no IP addresses beyond what's implicit in the HTTPS connection. Anthropic's API privacy policy applies to that data.

  • Calls go directly machine → api.anthropic.com
  • No proxy, no relay, no MeetBrief server in the path
  • Standard HTTPS — TLS encrypted in transit

Session history

If you enable session saving, transcripts and AI responses are written to your local userData directory. They are never uploaded anywhere. You can delete sessions from the History panel, or wipe all data from Settings → Clear All Data.

  • Saved locally in userData/sessions/
  • Never synced or uploaded
  • Fully deletable at any time from within the app
Open source

Don't take our word for it

Every line of MeetBrief is available on GitHub. The network calls are in src/utils/gemini.js and src/utils/cloud.js. The audio pipeline is in src/audio/. The storage layer is in src/storage.js.

Grep for fetch(, axios(, https.request(, or ws:// in the source to see every outbound network call. You'll find exactly three categories: the Anthropic API call (optional, uses your key), the Ollama localhost call (optional, stays local), and the Hugging Face model download (one-time, public weights only).
